# ADR 0001: Single-Tenant Pilot

## Status

Accepted

## Decision

The MVP will run as a single-tenant deployment for one institutional design partner.

## Rationale

- reduces build complexity
- avoids premature tenancy abstractions
- keeps audit, support, and operations narrow during the pilot

## Consequence

Multi-tenant scoping, self-serve provisioning, and tenant-aware billing are deferred to Beta / Real App phases.
